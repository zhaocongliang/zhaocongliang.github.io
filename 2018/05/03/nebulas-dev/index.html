Nebulas Dapp开发（七）- 网页钱包部署合约

2018-05-03 | 阅读：次

小亮在之前的星云链系列文章介绍了通过终端部署智能合约，接下来小亮带你通过网页版钱包部署合约，我们需要去下载网页钱包源码，以及chrome浏览器钱包插件。

网页钱包的使用

将网页钱包的源码下载到本地之后，解压，双击index.html

进入如下界面，页面的右上角可以进行中英文切换

来简单介绍一下网页钱包的功能

  • 新建钱包：新建钱包后注意保管好密码和私钥文件
  • 发送星云币：给任意其他的钱包地址发送星云币交易
  • 离线发送：离线发送星云币交易
  • 钱包信息：根据钱包地址查询钱包信息
  • 交易状态：查询一笔交易的状态
  • 智能合约：
    • 搜索合约：根据合约地址搜索合约信息
    • 部署合约：部署智能合约到公链或者测试链或私有链上
    • 执行合约：调用智能合约的功能

chrome浏览器钱包插件的使用

下载好钱包插件，解压WebExtensionWallet-master，然后点击chrome的”更多工具”–“扩展程序”进入如下界面，点击右上角的”开发者模式”–“加载已解压的扩展程序”，选择刚刚解压的钱包插件文件就可以安装了。

安装好后chrome上会有星云链钱包小图标，钱包插件功能这里就不介绍了，和网页版功能差不多。

部署Super Dictionary合约

首先Super Dictionary合约制作起来还是很简单的，我们只需三个步骤：

1 制作一个有文本框和按钮的Web页面。
2 制作一个只需要用JavaScript编写几十行代码的智能合约。
3 完成Web页面与智能合约的交互。

第一步，Web页面制作。 过程略过，简单讲一下交互方式。 网页的交互方式很简单：

（1）用户在输入框中输入要查询的词条。

（2）如果查询到结果，则返回该词条的详细内容。

（3）如果没有查询到，则允许用户添加释义。

第二步，编写智能合约

Super Dictionary智能合约所需要完成的业务逻辑并不复杂，尤其当开发语言是JavaScript的时候，只需几十行代码就可以完成，简单快捷。

本例中所涉及到的智能合约大体包括三个部分：

（1）词条数据结构

（2）查询词条的函数

（3）新增词条的函数

（1）编写词条数据结构

在这个项目的智能合约中，我们为每一个词条定义一个结构：包括作者(author)、词条名（key）、词条内容（value），代码如下：

"use strict";

var DictItem = function(text) {
	if (text) {
		var obj = JSON.parse(text);
		this.key = obj.key;
		this.value = obj.value;
		this.author = obj.author;
	} else {
	    this.key = "";
	    this.author = "";
	    this.value = "";
	}
};

（2）设置查询词条 & 新增词条：

星云链的智能合约运行环境内置了存储对象LocalContractStorage，可以存储数字，字符串，JavaScript对象，存储数据只能在智能合约内使用，其他合约不能读取存储的内容。所以查询词条就可以简化成了类似在map上对词条key值进行get操作。新增词条则演化为往map中进行put操作。 代码如下：

  save: function (key, value) {

        key = key.trim();
        value = value.trim();
        if (key === "" || value === ""){
            throw new Error("empty key / value");
        }
        if (value.length > 64 || key.length > 64){
            throw new Error("key / value exceed limit length")
        }

        var from = Blockchain.transaction.from;
        var dictItem = this.repo.get(key);
        if (dictItem){
            throw new Error("value has been occupied");
        }

        dictItem = new DictItem();
        dictItem.author = from;
        dictItem.key = key;
        dictItem.value = value;

        this.repo.put(key, dictItem);
    },

    get: function (key) {
        key = key.trim();
        if ( key === "" ) {
            throw new Error("empty key")
        }
        return this.repo.get(key);
    }

（3）部署合约

接下来小亮是把合约部署到本地的私有链上，所以先要启动本地私有链服务，然后开启一个矿工节点，这些也在之前的星云链系列教程中聊过了，这里就不重复了。 接下来点击网页钱包右上角切换到Localhost node，这个是本地私有链网络。然后点击"Contract"--"Deploy"，然后将合约的代码拷贝到"code:"中,如图 然后点击"select wallet file"选择本地私有链默认创建的21个钱包地址的私钥文件以及对应的输入密码，默认是"passphrase"，点击"unlock"，再点击"submit"，提交部署智能合约请求。 上图中txhash是部署这笔合约交易的哈希值，可以通过它查询到当前交易的状态，contract_address是智能合约地址，通过这个合约地址才能调用合约功能。 下图是根据txhash查询出来的交易状态信息：

至此我们已经将Super Dictionary智能合约部署到我们本地私有链中了，接下来就是通过前端页面去调用合约功能了。
